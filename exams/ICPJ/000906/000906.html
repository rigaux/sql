1   Langages de Requêtes (8 points)

Soit le schéma relationnel suivant avec trois relations :
  • Personnes(PNom, PVille, PDept, PTél) : cette relation contient les noms de personnes (PNom), leurs numéros de téléphone (PTel) et les villes (PVille) et départements (PDept) où ils habitent.
  • Hôtels(HNom, HVille, HDept, HTél, Cat, Prop) : le nom de l'hôtel (HNom), la ville (HVille) et le département (HDept), la catégorie (Cat) et le propriétaire (Prop).
  • Réservations(PNom, HNom, Début, Fin) : le nom de la personnes (PNom), le nom de l'hôtel (HNom) et le début (Début) et la fin (Fin) de la réservation.
Écrire les requêtes suivantes en algèbre relationnelle et en SQL :

Requête 1.1 :
(1 point) Les noms des personnes qui se trouvent à l'hôtel 'Château de Barive' le '15/07/2000'?

select distinct PNom

Requête 1.2 :
(1 point) Les noms des personnes qui ont réservé une chambre dans un hôtel du département où ils habitent?

Solution :

Requête 1.3 :
(1 point) Les hôtels qui n'ont pas de clients Parisiens?

Solution :
pHNom(Hotels) - pHNom(Reservations |><| sPVille='Paris'Personnes)
select HNom
  from Hôtels
 where not exists (select *
                     from Personnes, Réservations
                    where Personnes.PNom = Réservations.PNom
                      and Hôtels.HNom = Réservations.HNom
                      and PVille = 'Paris')
Requête 1.4 :
(1 point) Les hôtels qui n'ont que des clients Parisiens?

Solution :

Requête 1.5 :
(1 point) La requête précédente en calcul relationnel (domaine ou n-uplet)?

Solution :
{ h.HNom | Hotels(h) Ù ¬ $p, r: (Personnes(p) Ù Reservations(r) Ù
    p.PNom = r.PNom Ù h.HNom = r.HNom Ù p.PVille <> 'Paris' )}
Écrire les requêtes suivantes uniquement en SQL :
Requête 1.6 :
(1 point) Le nombre de réservations par hôtel (et le nom de l'hôtel) au mois de juillet (début de réservation au mois de juillet)?

Solution : 
select HNom, count(*)
  from Réservations
 where Début >= '01/07/2000' and Début <= '31/07/2000' 
 group by HNom;
Requête 1.7 :
(1 point) Les noms des hôtels avec au moins trois clients de la même ville (et le nom de la ville)?

Solution : 
select HNom, PVille
  from Personnes, Réservations
 where Personnes.PNom = Réservations.PNom
 group by HNom, PVille
having count(*) >= 3;

2   Dépendances fonctionnelles (3 points)

Pour chaque relation de la section précédente on a défini un ensemble de dépendances fonctionnelles (on a remplacé les noms d'attributs par leurs initiales) :

Personnes(PN, PV, PD, PT) FPersonnes={PN ® (PV, PD, PT); PT ® PN}
Hôtels(HN, HV, HD, HT, C, P) FHotels={HN ® (HV, HD, HT, C)}
Réservations(PN, HN, D, F) FReservations={}
Question 2.1 :
(1 point) Quelles sont les clés des trois relations?

Solution :
  1. Personnes: deux clés: PN et PT
  2. Hôtels: une clé: (HN, P)
  3. Réservations: une clé: (PN, HN, D, F)

Question 2.2 :
(0,5 points) En regardant les dépendances fonctionnelles, est-ce que chaque hôtel ne peut avoir qu'un propriétaire?

Solution : NON

Question 2.3 :
(1,5 points) Est-ce que la relation Hôtels est en 3e Forme Normale? Justifiez votre réponse.

Solution : NON; par exemple, dans la DF HN ® HV (après la décomposition), HN n'est pas une super-clé et HV n'appartient pas à une clé.

4   Concurrence (5 points)

H: r1[x] r2[x] w2[y] r3[y] w3[z] w1[z] w2[x] r3[x] c2 c1 w3[x] c3
