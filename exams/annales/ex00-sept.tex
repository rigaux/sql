\vspace*{0.5cm}

Une startup décide de créer un site web proposant tous les restaurants
parisiens. Le site est construit sur une base de données
contenant les restaurants, les plats proposés et les menus.
Voici les tables.  Comme d'habitude, les clés étrangères 
ne sont pas représentées\,:
      \begin{itemize}
        \item {\tt Restaurant (\underline{nomRestau}, arrondissement)}
         \item {\tt Plat (\underline{nomRestau, nomPlat}, prix, typePlat)}
         \item {\tt Menu (\underline{nomRestau, noMenu}, nom, prix)}
         \item {\tt DétaiMenu (\underline{nomRestau, noMenu, nomPlat})} 
       \end{itemize}

L'attribut {\tt arrondissement} contient le numéro
de l'arrondissement du restaurant.
La table {\tt Plat} donne l'ensemble des plats proposés par un restaurant.
Le type du plat est soit {\tt Entrée}, soit {\tt Plat principal},
soit {\tt Dessert}.
La table {\tt DétailMenu} donne les plats composant un menu\,:
un ensemble d'entrées, de plats principaux et de dessert.

\smallskip
{\bf Requ\^etes ({\bf 10 pts})}

Exprimez en SQL les requ\^etes suivantes:

\begin{enumerate}
  \item  ({\bf 1 pt}) Nom des restaurants du douzième arrondissement.

  \item  ({\bf 1 pt}) Nom des restaurants dont le nom
      contient le mot <<\,fourchette\,>>.

  \item  ({\bf 2 pts})  Quels 
               restaurants (nom et arrondissement)
                 proposent des menus à 150 frs\,?
  
  \item  ({\bf 2 pts}) Donnez la liste des entrées du menu numéro 2 du restaurant
    <<\,Boboche\,>>.

  \item  ({\bf 2 pts}) Donnez les menus (nomRestau, noMenu)
               ayant un plat plus cher que le prix du menu lui-même.

  \item  ({\bf 2 pts}) Quels restaurants (nomRestau) ont un menu qui ne propose
    pas de dessert\,?

  \end{enumerate}

\smallskip
{\bf Schéma (10 pts)}

\begin{enumerate}
   \item ({\bf 3 pts}) Donnez les commandes {\tt CREATE TABLE} d\'ecrivant
         le sch\'ema des relations {\tt Plat}, {\tt Menu} et
            {\tt DetailMenu}. Indiquez 
         les cl\'es \'etrang\`eres et les cl\'es primaires.

   \item ({\bf 1 pts}) Donnez la contrainte SQL2 qui force
      le type d'un plat aux trois valeurs possibles.
 
   \item ({\bf 2 pts})
       On veut permettre aux internautes de noter les restaurants.
       Pour cela on veut connaître les internautes (nom, prénom, email)
       et savoir que tel internaute a donné telle note à tel restaurant.

       Donnez le schéma E/A correspondant à l'ajout de ces informations.

   \item  ({\bf 2 pts}) Donnez le schéma des nouvelles tables à créer.

    \item ({\bf 2 pts})
           Quel est l'ordre SQL donnant la moyenne des notes obtenues
        par <<\,Boboche\,>>\,? 
\end{enumerate}
