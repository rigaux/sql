Examen Bases de Données
Mercredi 14 juin 2000 Solution sur le WEB, et au secrétariat




Soit P la note du partiel, et E la note de l'examen (pour la partie ``bases de données''). La note finale en base de données sera alors: Max (E, (E+P)/2).




La note finale de l'UV est la moyenne de la note en base de données et de la note en processus d'informatisation.




Les informations auxquelles on s'intéresse sont les droits d'accès à une base de données, tels qu'ils peuvent etre définis par la commande GRANT de SQL2. On veut définir un modèle de représentation de ces droits d'accès, les stocker dans une base de données, et les interroger avec SQL. Voici les principales notions de l'application (elles doivent vous etre déjà familières).
  • Les tables sont des tables relationnelles, avec quelques informations générales comme le nom de la table, le nom de l'utilisateur qui a créé la table, etc.

  • Les utilisateurs sont caractérisés par leur ID, leur nom et leur prénom.

  • Les privilèges définissent les droits des utilisateurs sur les tables. Les droits sont les types d'action que l'on peut effectuer sur une table.

  • Un programme est exécuté par un utilisateur et accède à la base. Un programme a les mêmes privilèges que l'utilisateur qui l'exécute.
Voici le schéma de la base. Comme d'habitude, les clés étrangères ne sont pas représentées :
  • Table (nomTable, idCreateur)
  • Utilisateur (id, nom, prenom)
  • Privilege (id, nomTable, idUtilisateur, droit)
  • Programme (idUtilisateur, nomProg)

Schéma
  1. (2 pts) Donnez les commandes CREATE TABLE décrivant le schéma des relations ci-dessus. Indiquez soigneusement les clés étrangères et les clés primaires.

  2. (2 pts) Donnez la contrainte SQL2 qui force l'attribut droit à prendre ses valeurs dans l'ensemble {SELECT, INSERT, UPDATE, DELETE, ALL}. De plus il faut toujours, pour un privilège, connaitre la valeur de droit, nomTable et idUtilisateur.


    Solution :

    Exemple pour la relation Privilege : 
    
CREATE TABLE Privilege 
   (id NUMBER (10),
    nomTable    VARCHAR (30),
    idUtilisateur  NUMBER (10),
    droit VARCHAR (10) CHECK (droit IN 
    ('SELECT','INSERT','DELETE','UPDATE','ALL')),
     PRIMARY KEY (id) NOT NULL,
     FOREIGN KEY (nomTable) REFERENCES Table,
     FOREIGN KEY (idUtilisateur) REFERENCES Utilisateur);



  3. (2 pts) Donnez le schéma E/A correspondant au schéma relationnel.

  4. (2 pts) Supposons que l'on sache à quelles tables accède un programme. Indiquez comment on représente cette information dans le schéma E/A, puis dans le schéma relationnel.


    Solution :

    Il faut ajouter une association Acces n-m entre Programme et Table, ce qui donne la table : 
    
Acces (nomTable, idUtilisateur, nomProg)
    Les trois champs constituent la clé.



Requêtes (8 pts)

Sur le schéma donné dans l'énoncé, exprimez en SQL les requêtes suivantes:
  1. Nom des tables pour lesquelles un utilisateur a le privilège 'ALL'


    Solution :

    SELECT nomTable FROM Privilege WHERE droit = 'ALL'




  2. Liste des programmes de l'utilisateur 'Gates'


    Solution :

    SELECT nomprog FROM Programme p, Utilisateur u WHERE nom='Gates' AND p.idUtilisateur=u.id




  3. Nom et prénom des utilisateurs qui peuvent détruire dans la table 'Film'.


    Solution :

    SELECT nom, prenom FROM Utilisateur u, Privilege p WHERE nomTable='Film' AND droit='DELETE' AND u.id = p.idUtilisateur




  4. Nom des tables sur lesquelles l'utilisateur 20 a un droit de mise-à-jour. Rappel: mise-à-jour signifie INSERT ou UPDATE ou DELETE ou ALL.


    Solution :

    SELECT nomTable FROM Privilege WHERE idUtilisateur=20 AND droit IN ('ALL', 'INSERT', 'DELETE', 'UPDATE')




  5. Donnez les tables pour lesquelles le créateur de la table n'a pas le droit 'DELETE'.


    Solution :

    SELECT nomTable FROM Table t WHERE idCreateur NOT IN (SELECT idUtilisateur FROM Privilege p WHERE droit='DELETE' AND p.nomTable=t.nomTable)




  6. Quelles sont les tables sur lesquelles au moins 4 utilisateurs ont le droit 'ALL' ?


    Solution :

    SELECT nomTable FROM Privilege WHERE droit='ALL' GROUP BY nomTable HAVING COUNT(*) >= 4



Requetes imbriquées (4 pts)

Voici 3 ordres SQL. 
SELECT nomTable FROM   Utilisateur u, privilege p
WHERE  u.id = p.idUtilisateur
AND    nom = 'Rigaux' AND    droit = 'ALL'

SELECT nomTable FROM  privilege p
WHERE  p.idUtilisateur = (SELECT id FROM utilisateur
                          WHERE nom = 'Rigaux')
AND    NOT (droit != 'ALL')

SELECT nomTable FROM  privilege p
WHERE  NOT EXISTS (SELECT 1 FROM utilisateur
                    WHERE nom != 'Rigaux' 
                    AND p.idUtilisateur=id)
AND    droit = 'ALL'
  1. Exprimez, en une phrase, ce que fait le premier ordre.
  2. Parmi les 2 autres requetes, lesquelles sont équivalentes (elles donnent le meme résultat) à la requete 1 ? Exprimez, en une phrase, ce que font celles qui ne sont pas équivalentes.

Solution :

  1. La requete sélectionne les tables sur lesquelles Rigaux a le privilège 'ALL'

  2. La première est équivalente à la deuxième. La troisième séléctionne les tables sur lesquelles (1) il y a un privilège 'ALL', (2) seul Rigaux a des privilèges. Ce n'est pas la meme chose.
