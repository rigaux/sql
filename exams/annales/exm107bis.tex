   \item (1 point) Combien de pages le SGBD doit-il lire pour trouver les

     Indiquez précisément les pages lues.

    \item (1 point)

   structure combien de lectures faut-il pour trouver l'enregistrement 12\,?

   \end{enumerate}

\section{Optimisation (9 points)}

Soit la requête SQL suivante, calculant l'âge des personnes de
strictement plus de 40 ans ayant répondu "oui" à la question "baisse
des impôts".

\begin{smallverbatim}
select age from PERSONNE, QUESTION, AVIS
where AVIS.numPers=PERSONNE.numPers
and AVIS.numQ=QUESTION.numQ
and AVIS.reponse="oui"
and PERSONNE.age > 40
and QUESTION.description="baisse des impôts".
\end{smallverbatim}

\begin{enumerate}
 \item (2 pts) Donnez (sous la forme de votre choix\,: expression algébrique,
   arbre d'exécution, EXPLAIN Oracle)
  le plan d'exécution qui vous semble le meilleur pour cette requête,
   en expliquant vos choix.

  \item On considère la requête SQL suivante :
\begin{smallverbatim}
select PERSONNE.numPers,reponse from PERSONNE, AVIS
where AVIS.numPers=PERSONNE.numPers
and PERSONNE.age > 40
\end{smallverbatim}

Voici le plan d'exécution EXPLAIN donné par le SGBD :
\begin{smallverbatim}
Execution Plan
----------------------------------------------------------
0 SELECT STATEMENT
1   NESTED LOOPS
2     TABLE ACCESS (BY ROWID) PERSONNE
3       INDEX (RANGE SCAN) PERSONNE_IDX
4     TABLE ACCESS (BY ROWID) AVIS
5       INDEX (RANGE SCAN) AVIS_IDX
\end{smallverbatim}

\begin{enumerate}
  \item  (1 point) Indiquez pour chaque relation si un index est utilisé 
    et, le cas échéant, sur quel attribut porte l'index.

 \item (2 points) Expliquez le fonctionnement de ce plan d'exécution, 
      en précisant les algorithmes utilisés.
\end{enumerate}

  \item (2 points) On effectue une jointure  entre \textit{Avis}
et \textit{Personne}. \textit{Avis} occupe 1\,000 blocs,
    \textit{Personne} 3\,000 blocs, et \textit{Personne}
     est indexée par un arbre B+ ayant 3 niveaux. Notez que
   pour un tuple dans \textit{Avis}, on cherche un tuple
     dans \textit{Personne} et un seul.

   Indiquez dans le pire des cas le nombre de lectures pour
   un algorithme par boucles imbriquées
      indexées. 

   \item (2 pts) On dispose d'une mémoire centrale de 1\,500 blocs. Quel
   algorithme est préférable aux boucles imbriquées indexées, et pourquoi\,?
\end{enumerate}

\section{Concurrence (5 points)}

L'exécution suivante est reçue par le système de l'institut de sondage :
$$H : r_1[x] r_2[y] w_1[x] r_3[y] r_2[x] w_3[y] r_2[z] c_1 r_3[z] w_2[z] c_2 w_3[z] c_3$$

\begin{enumerate}
  \item  (1 point) Parmi les programmes qui s'exécutent dans le système, 
il y a \textit{ModifierAvis(description\_question, avis\_donné,
nouvel\_avis)}, 
qui modifie pour la question \textit{description\_question} seulement 
les avis \textit{avis\_donné} en \textit{nouvel\_avis}. Si
les enregistrements de $H$ sont des nuplets des relations de la base 
de données, montrez (en justifiant votre réponse)
quelles transactions de $H$ pourraient provenir de \textit{ModifierAvis}.

On suppose qu'une description de question donne accès direct aux 
nuplets QUESTION qui lui correspondent. Aussi, un numéro de question 
donne accès direct aux nuplets AVIS sur la question.

  \item (1 point) Vérifiez si $H$ est sérialisable en identifiant 
les conflits et en construisant le graphe de sérialisation.

  \item (1 point) Montrez que $H$ n'évite pas les annulations en cascade 
 à cause de la position de l'un des \textit{Commit}.
 
  \item (2 points) Quelle est l'exécution obtenue par verrouillage 
 à deux phases à partir de $H$? 
 Le relâchement des verrous d'une transaction se fait au 
   \textit{Commit} et à ce moment on exécute en priorité les opérations
bloquées en attente de verrou, dans l'ordre de leur blocage.
   
\end{enumerate}
