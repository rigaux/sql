Les informations auxquelles on s'intéresse sont les 
droits d'accès à une base de données, tels qu'ils
peuvent etre définis par la commande {\tt GRANT}
de SQL2. On veut définir un modèle de représentation
de ces droits d'accès, les stocker dans une base
de données, et les interroger avec SQL.  
Voici les principales notions de l'application
(elles doivent vous etre déjà familières).

\begin{itemize}
   \item {\bf Les tables}  sont des tables relationnelles,
     avec quelques informations générales comme le nom
    de la table, le nom de l'utilisateur qui a créé
    la table, et leur attributs.

   \item {\bf Les utilisateurs} sont caractérisés
             par leur ID, leur nom et leur prénom.

   \item {\bf Les privilèges} définissent les {\bf droits}
      des {\bf utilisateurs} sur les {\bf tables}. 
   {\bf Les droits} sont les types d'action
    que l'on peut effectuer sur une table.

\end{itemize}

Voici le schéma de la base. Comme d'habitude, les clés étrangères 
ne sont pas représentées\,:

      \begin{itemize}
        \item  Table (\textbf{nomTable}, idCreateur)
        \item  Attribut  (\textbf{nomTable}, \textbf{nom}, type, longueur)
         \item  Utilisateur (\textbf{id}, nom, prenom)
         \item  Privilège (\textbf{id}, nomTable, idUtilisateur, droit)
       \end{itemize}

\section{Schéma (\textbf{6 pts})}

\begin{enumerate}
   \item ({\bf 2 pts})
       Donnez le schéma E/A correspondant au schéma relationnel.

   \item ({\bf 2 pts}) Donnez les commandes {\tt CREATE TABLE} décrivant
         le schéma des relations ci-dessus. Indiquez soigneusement
         les clés étrangères et les clés primaires.

   \item ({\bf 1 pt}) Donnez la contrainte SQL2 qui force
      l'attribut {\tt droit} à prendre ses valeurs
      dans l'ensemble {\tt \{SELECT, INSERT, UPDATE, DELETE, ALL\}}.
      De plus il faut {\bf toujours}, pour un privilège,
         connaitre la valeur
       de {\tt droit}, {\tt nomTable} et {\tt idUtilisateur}.
       
\begin{solution}
Exemple pour la relation {\tt Privilege}\,:
\begin{verbatim}
CREATE TABLE Privilege 
   (id NUMBER (10),
    nomTable    VARCHAR (30),
    idUtilisateur  NUMBER (10),
    droit VARCHAR (10) CHECK (droit IN 
    ('SELECT','INSERT','DELETE','UPDATE','ALL')),
     PRIMARY KEY (id) NOT NULL,
     FOREIGN KEY (nomTable) REFERENCES Table,
     FOREIGN KEY (idUtilisateur) REFERENCES Utilisateur);
\end{verbatim}

\end{solution}


   \item  ({\bf 1 pt}) On veut enregistrer les informations
             sur les {\em sessions} utilisateur. Une session
                     est caractérisée par son instant de début, son
                instant de fin et l'utilisateur qui est connecté.
              De plus, pendant une session, on lit ou on met à jour
            certaines tables.   Indiquez
      comment on représente cette information dans le schéma E/A,
     puis dans le schéma relationnel. 

\begin{solution}
Il faut ajouter une association {\tt Acces} n-m entre {\tt Programme}
et {\tt Table}, ce qui donne la table\,:
\begin{verbatim}
Acces (nomTable, idUtilisateur, nomProg, typeAccès)
\end{verbatim}
Les trois premiers champs constituent la clé.
\end{solution}
\end{enumerate}
    
\section{Requêtes ({\bf 8 pts})}

Sur le schéma donné dans l'énoncé,
exprimez en algèbre, puis en SQL les requêtes suivantes:

\begin{enumerate}
  \item Nom des tables pour lesquelles un utilisateur
     a le privilège '\texttt{ALL}'
\begin{solution}
{\tt SELECT nomTable FROM Privilege WHERE droit = 'ALL'}
\end{solution}

  \item Liste des attributs de type \texttt{VARCHAR(30)}.

\begin{solution}
{\tt SELECT nomprog FROM Programme p, Utilisateur u
WHERE nom='Gates' AND p.idUtilisateur=u.id}
\end{solution}

  \item Quel utilisateur a créé une table avec un attribut de 
    type \texttt{BLOB}\? Donnez le nom de l'utilisateur et le nom
        de la table.

  \item Nom et prénom des utilisateurs qui peuvent détruire
     dans la table 'Film'.
\begin{solution}
{\tt SELECT nom, prenom FROM Utilisateur u, Privilege p
WHERE nomTable='Film' AND droit='DELETE' 
AND  u.id = p.idUtilisateur}
\end{solution}

  \item Paires d'utilisateurs (nom et prénom pour chacun)
        qui ont tous deux un même droit sur une même table.

  \item Donnez les tables (nom et id du créateur)
         pour lesquelles le créateur
     de la table n'a pas le droit \texttt{DELETE}.
\begin{solution}
{\tt SELECT nomTable FROM Table t
WHERE idCreateur NOT IN (SELECT idUtilisateur 
                         FROM   Privilege p
                         WHERE  droit='DELETE'
                         AND    p.nomTable=t.nomTable)
}
\end{solution}
\end{enumerate}

Exprimez les requêtes suivantes seulement en SQL.

\begin{enumerate}
  \item Nom des tables sur lesquelles l'utilisateur 'Bill Gates' a un droit
de mise à jour. Rappel: mise à jour signifie \texttt{INSERT} ou 
    \texttt{UPDATE}    ou \texttt{DELETE} ou \texttt{ALL}.

\begin{solution}
{\tt SELECT nomTable FROM Privilege 
WHERE idUtilisateur=20
AND droit IN ('ALL', 'INSERT', 'DELETE', 'UPDATE')
}
\end{solution}

  \item Nom des utilisateurs qui ont créé une table 
               sans attribut de type \texttt{VARCHAR}.  

  \item Combien de tables ont été créées par l'utilisateur Rigaux\,?

  \item Quelles sont les tables ayant au moins 4 attributs
             de type \texttt{INT}\,?

\begin{solution}
{\tt SELECT nomTable FROM Privilege WHERE droit='ALL'
     GROUP BY nomTable
     HAVING COUNT(*) >= 4
}
\end{solution}

  \end{enumerate}

\section{Requêtes imbriquées ({\bf 2 pts})}

Voici 3 ordres SQL.

\begin{verbatim}
SELECT nomTable FROM   Utilisateur u, privilege p
WHERE  u.id = p.idUtilisateur
AND    nom = 'Rigaux' AND    droit = 'ALL'
\end{verbatim}

\begin{verbatim}
SELECT nomTable FROM  privilege p
WHERE  p.idUtilisateur = (SELECT id FROM utilisateur
                          WHERE nom = 'Rigaux')
AND    NOT (droit != 'ALL')
\end{verbatim}

\begin{verbatim}
SELECT nomTable FROM  privilege p
WHERE  NOT EXISTS (SELECT 1 FROM utilisateur
                    WHERE nom != 'Rigaux' 
                    AND p.idUtilisateur=id)
AND    droit = 'ALL'
\end{verbatim}

\begin{enumerate}
  \item Exprimez, en une phrase, ce que fait le premier ordre.
  \item Parmi les 2 autres requêtes, lesquelles sont {\bf équivalentes}
      (elles donnent le même résultat, quelle que soit l'instance) 
      à la requête 1\,? 
       Exprimez, en une phrase, ce que font celles qui
   ne sont pas équivalentes.
\end{enumerate}

\begin{solution}
   \begin{enumerate}
      \item La requete sélectionne les tables sur lesquelles
     Rigaux a le privilège 'ALL'

       \item La première est équivalente à la deuxième.
        La troisième séléctionne les tables
         sur lesquelles (1) il y a un privilège 'ALL', 
         (2) seul Rigaux a des privilèges. Ce n'est pas
      la meme chose.
    \end{enumerate}
\end{solution}

\begin{enumerate}
  \item On suppose que les seuls types connus sont
         \texttt{INT} et \texttt{VARCHAR}. Pour ce dernier
           la longueur est représentée par l'attribut
            \texttt{longueur} dans la table \textit{Attribut},
            sinon \texttt{longueur} est à \texttt{NULL}.

          et longueur, selon le format suivant\,:
\begin{verbatim}
<nomTable>, créée par <nomCréateur>
   <nomAttribut> [INT | VARCHAR(<longueur>)]
   ...
\end{verbatim}

  \item \'Ecrire une procédure qui affiche le schéma de
        de toutes
        les tables créées par l'utilisateur Rigaux (bien entendu
             vous réutiliserez la procédure précédente).
\end{enumerate}
