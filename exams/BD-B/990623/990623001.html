1  Modčle Relationnel (10 points)

L'usine de production de poulets LPH (La Poule Heureuse) gčre son stock d'aliments pour les poulets avec Oracle. Voici le schéma (dans chaque relation, excepté Fournisseur, les attributs formant une clé sont soulignés) :

  • Container(IdC, NomF, TypeA) : IdC est l'identificateur du container, NomF est le nom du fournisseur et TypeA est le type d'aliment ('farine de poisson', 'farine animale', 'tourteau de soja') contenu dans le container.
  • Fournisseur(NomF, SiteF, TypeA, PrixA) : NomF est le nom du fournisseur, SiteF est le site (la ville) de production du fournisseur (un site par fournisseur), TypeA est le type d'aliment fourni par le fournisseur et PrixA est le prix par kilo de type d'aliment chez le fournisseur donné.
  • Consommation(IdC, DateC, QuanC) : IdC est l'identificateur du container, DateC est la date de consommation et QuanC est la quantité consommée (en kilos) ŕ cette date.
  • Livraison(IdC, DateL, QuanL) : IdC est l'identificateur du container, DateL est la date de livraison et QuanL est la quantité livrée (en kilos) dans le container.

1.1  Conception de schéma (3 points)

  1. Est-ce qu'un container peut ętre livré plusieurs fois ? (1 point) Justifiez votre réponse.
  2. Les dépendances fonctionnelles de la relation Fournisseur sont F={NomF -> SiteF, (NomF, TypeA) -> PrixA}. Quelle hypothčse a été prise pour justifier qu'on n'a pas la dépendance NomF -> (SiteF, TypeA) ? (0.5 point). Montrez que cette relation n'est pas en 3e Forme Normale et faites une décomposition en relations en 3e Forme Normale, sans perte d'information (SPI) et qui préserve les dépendances fonctionnelles (PDF). (1.5 points)

  FROM Livraison, Container, Consommation

    Solution :

        Container(I, NF, 'farine animale') /\
        Consommation(I, DC, QC) /\ DC > '03/06/99')}



  4. Noms de fournisseur qui ne fournissent pas de farines animales. (1 point)

    Solution :

    TypeA='farine animale'
    		Fournisseur)

  FROM Fournisseur

                      FROM Fournisseur

  5. Exprimer la requęte précédente dans le calcul nuplet (1 point).

Écrire les requętes suivantes seulement en SQL :
  1. Noms des fournisseurs, types d'aliment et total de la quantité d'aliments livrés, classés par fournisseur et par type d'aliment. (1 point)

    Solution : 
    SELECT NomF, TypeA, SUM(QuanL)
  FROM Livraison, Container
 WHERE Livraison.IdC = Container.IdC
GROUP BY NomF, TypeA;
  2. Noms des fournisseurs qui ont livré plus que 5 tonnes de farines animales aprčs le 6 juin. (1 point)

    Solution : 
    SELECT NomF
  FROM Livraison, Container
 WHERE Livraison.IdC = Container.IdC
   AND DateL > '6/6/99'
   AND TypeA = 'farine animale'
GROUP BY NomF
HAVING SUM(QuanL) > 5000;
