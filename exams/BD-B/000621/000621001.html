1   Langages de Requêtes (8 points)

Le fournisseur d'accès à Internet LibertyWorld utilise Oracle pour gérer les informations concernant les Internautes abonnés. Voici le schéma relationnel utilisé :

  • Abonnés(Login, Nom, Prénom, NumTél, Ville) : Login est un login unique attribué par le fournisseur d'accès à chaque utilisateur. Nom, Prénom, NumTél et Ville correspondent au nom, prénom, numéro de téléphone et à la résidence de l'abonné.
  • Abonnements(Login, TypeAbonnement, EspaceMax, AdressesMax, EspaceUtilisé, AdressesUtilisées) : TypeAbonnement est le type d'abonnement (Gratos, WeekEnd, DeLuxe, ...) choisi par le client. EspaceMax et AdressesMax correspondent à l'espace maximale (en MO) et le nombre d'adresses email maximal pour chaque type d'abonnement (e.g. TypeAbonnement = 'Gratos' : EspaceMax = 5 et AdressesMax = 10). EspaceUtilisé et AdressesUtilisées correspondent à l'espace disque (en MO) et le nombre d'adresses email utilisées par le client.
  • Connexions(Login, DAte, Heure, Durée) : la table qui contient des informations sur les connexions pour chaque login : Date, Heure, et Durée correspondent à la date (format 'jj/mm/aaaa'), l'heure (format 'hh:mm') et la durée (en minutes) de chaque connexion effectuée par l'abonné.
Écrire les requêtes suivantes en algèbre relationnelle et en SQL :
Requête 1.1 :
(1 point) Les noms des abonnés parisiens avec un abonnement Gratos  ?

Solution :
pNom(sVille = 'Paris'(Abonnes) |><| sTypeAbonnement = 'Gratos'(Abonnes))
select Nom
  from Abonnes, Abonnements
 where Abonnes.Login = Abonnements.Login
   and Ville = 'Paris'
   and TypeAbonnement = 'Gratos';
Requête 1.2 :
(2 points) Les noms des abonnés qui se sont connectés le 1er Mai 2000 et qui utilisent plus d'espace disque qu'autorisé par le type de leur abonnement ?

Solution :
pNom(Abonnes |><| sDate = 01/05/2000 Connexions |><| sEspaceUtilise > EspaceMax Abonnements)
select Login
  from Abonnes, Connexions, Abonnements
 where Abonnes.Login = Connexions.Login
   and Abonnes.Login = Abonnements.Login
   and EspaceUtilise > EspaceMax
   and Date = '01/05/2000';
Requête 1.3 :
(1 point) La requête précédente en calcul relationnel (domaine ou n-uplet).

Solution :

{ L | $ N,P,NT,V,TA,EM,AM,H,D,EU,AU (Abonnes(L,N,P,NT,V) Ù     (1)
    Abonnements(L,TA,EM,AM,EU,AU) Ù     (2)
    Connexions(L,'01/05/2000',H,D) Ù EU > EM) }     (3)

Requête 1.4 :
(2 points) Les logins des abonnés qui ne se sont pas connectés depuis le 1er Mai 2000 ?

Solution :

pLogin(Abonnes) - pLogin(sDate >= '01/05/2000' Connexions)

select Login
  from Abonnes
 where Login not in (select Login
                       from Connexions
                      where Date >= '01/05/2000');
Écrire les requêtes suivantes uniquement en SQL :
Requête 1.5 :
(1 point) La durée totale des connexions depuis le 1er Mai 2000 par type d'abonnement ?

Solution : 
select TypeAbonnement, Sum(Durée)
  from Abonnement, Connexions
 where Date >= '01/05/2000'
   and Abonnement.Login = Connexions.Login
 group by TypeAbonnement;
Requête 1.6 :
(1 point) Le(s) login(s) avec le plus important temps total de connexion (temps total de connexion maximum) ?

Solution : 
select Login
  from Connexions
 group by Login
having Sum(Durée) = (select Max(Durée)
                       from Connexions
                      group by Login);
