2  Optimisation des requêtes SQL (3 points)

Étant donné le schéma de la Section 1,

  • soit la requête suivante: 
    select NomC
  from Clients, Voyages, Visites
 where Prix < 5000
   and EndV = 'parc Kruger'
   and Clients.NumV = Voyages.NumV
   and Visites.NumV = Voyages.NumV;
    Supposons que NumV est une clé primaire de la relation Voyages et qu'il n'y a pas d'autres clés. Comment cette requête pourrait être évaluée? Vous pouvez utiliser un plan d'exécution généré par l'outil EXPLAIN d'Oracle pour donner les détails nécessaires à la compréhension. (1 point)

    Solution:

    Plan d'execution
--------------------------------------------------------------------------
0 SELECT STATEMENT
  1 MERGE JOIN
    2 SORT JOIN
      3 NESTED LOOPS
        4 TABLE ACCESS FULL VISITES
        5 TABLE ACCESS BY ROWID VOYAGES
          6 INDEX UNIQUE SCAN SYS_C002640
    7 SORT JOIN
      8 TABLE ACCESS FULL CLIENTS
  • soit la requête suivante: 
    select A.EndV
  from Visites A, Visites B
 where A.NumV <> B.NumV
   and A.DatV = B.DatV
   and A.EndV = B.EndV
    L'outil EXPLAIN de Oracle fournit le plan d'exécution suivant: 
    Plan d'execution
----------------------------------------------------------------------
0 SELECT STATEMENT
  1 NESTED LOOPS
    2 TABLE ACCESS FULL VISITES
    3 TABLE ACCESS BY ROWID VISITES
      4 AND-EQUAL
        5 INDEX RANGE SCAN IND1
        6 INDEX RANGE SCAN IND2
    1. Sur quelle relation et quel attribut les index IND1 et IND2 sont-ils définis pour obtenir ce plan d'exécution? (1 point)

      Solution:

      IND1 et IND2 sont définis sur les attributs DatV et EndV de la relation Visites.
