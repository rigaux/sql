\section{Concurrence (5 points)}

Voici une procédure de chargement d'une photo  pour un propriétaire nommé $n$.

\begin{small}
\begin{verbatim}
Procédure charger(Photo photo, String n, Statut s)
début
  // Trouvons la date du jour
  d := dateDuJour()
  // Cherchons le propriétaire
  p := select * from Propriétaire where nom = :n
  // Si on l'a trouvé: mise ŕ jour
  if (p != null) alors
    update Propriétaire set nombrePhotosPubliées = :p.nombrePhotosPubliées + 1
    insert into Photo values (:photo)
    commit
  fait

   insert intoPublication values (:photo.id, :d, :s)  
   commit
fin
\end{verbatim}
\end{small}

\begin{enumerate}
	\item (1,5 point) Critiquez cette procédure, et notamment l'emplacement des \texttt{commit}. Signalez 
	deux cas d'anomalie et proposez une correction de la procédure.
	\item (1 point) On exécute deux fois la procédure avec 
	\texttt{charger (p1, n, 'public')}  et \texttt{charger (p2, n, 'public')}. On obtient
	l'exécution concurrente suivante ($u_1$ et $u_2$ désignent les lignes de \texttt{Publication} créées
	pendant l'exécution):
	
	$$H = r_1(n) r_2(n) w_1(n) w_1(p_1) w_2(n) w_2(p_2) w_2(u_2) w_1(u_1) C_1 C_2$$
	
    Citer un exemple d'anomalie entrainée par cette exécution.
    
    \item (1 pt) Montrer que l'exécution précédente n'est pas sérialisable.
    
    \item (1,5 pt) Appliquer l'algorithme de verrouillage ŕ deux phases ŕ $H$.
\end{enumerate}


\begin{correction}

\begin{enumerate}
  \item Le premier \texttt{commit} est effectué  alors que la transaction n'est pas complčte. 
    \begin{itemize}
       \item Si plantage aprčs le premier \texttt{commit}: la photo est insérée mais
           pas la date de publication.
        \item Si pas de propriétaire: on insčre quand męme une date de publication et on valide.
    \end{itemize}
     Il faut ajouter le second \texttt{insert} dans le corps du test, avant le premier \texttt{commit},
       et supprimer le second \texttt{commit}.
     \item Clairement la mise ŕ jour de la seconde transaction écrase la premičre: le nombre
        de photos augmente de 1 pour 2 photos insérées.
        \item Conflits $r_2(n) \to w_1(n)$ et  $w_1(n) \to w_2(n)$, donc non sérialisable.
        \item Cas typique oů on aboutit ŕ un interblocage.  
\end{enumerate}
\end{correction}
