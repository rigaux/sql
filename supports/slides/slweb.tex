\begin{slide}
\subsection{Pourquoi un SGBD dans un site WEB\,?}

Un site WEB = un ensemble de pages HTML. Inconvénients\,:

\begin{enumerate}
  \item Les pages HTML sont {\bf statiques}
  \item Beaucoup d'information\,? Beaucoup de pages HTML\,!
\end{enumerate}

Avec un SGBD:
\begin{itemize}
  \item On définit quelques squelettes de pages HTML.
  \item On les remplit {\bf dynamiquement} avec des données
       selon les besoins.
\end{itemize}

Inconvénient\,: cela complique l'architecture.
\end{slide}

\begin{slide}
\subsection{Deux architectures types}

L'architecture de type CGI\,:
\begin{itemize}
  \item On ajoute un serveur applicatif au démon {\tt httpd}
  \item On utilise des formulaires HTML pour appeler le serveur
       applicatif et pour lui passer des paramčtres.
\end{itemize}

L'architecture de type JDBC\,:
\begin{itemize}
  \item On ajoute une couche SQL du côté client.

  \item Cette couche dialogue directement avec le serveur du SGBD.
\end{itemize}
\end{slide}

\begin{slide}
\subsection{Quelques éléments de comparaison}

Avec le CGI\,:
\begin{itemize}
  \item Le client reste du pur HTML. Il est ``léger''\,!
  \item Tout le service est du côté serveur\,: beaucoup de trafic réseau.
  \item Communication entre client et serveur assez compliquée.
\end{itemize}

Avec le JDBC
\begin{itemize}
  \item Une partie de la logique de l'application est sur le client. 
       Il est ``gras''\,! (Plus ou moins ...)

  \item Beaucoup plus de possibilités pour l'interface utilisateur.

   \item Meilleure intégration interface/application
\end{itemize}

NB\,: d'autres architectures (variantes) possibles.

\end{slide}

\begin{slide}
\subsection{Comment \c{c}a marche\,?}

Quand on appuie sur le bouton, le navigateur\,:
\begin{enumerate}
  \item récolte les valeurs saisies dans les champs,

  \item place ces valeurs dans une chaîne de caractčres.

  \item demande au {\tt httpd} d'exécuter le programme {\tt cgifilm}
\end{enumerate}
Le client\,:
\begin{enumerate}
  \item Place la chaîne contenant les paramčtres dans une variable
       d'environnement.
  \item Exécute le programme qui doit récupérer la variable et la décrypter.

   \item Ecrit le résultat (un fichier HTML) sur sa sortie standard.
\end{enumerate}

\end{slide}

\begin{slide}
\subsection{JDBC}

Important\,: un programme JDBC est indépendant de tout SGBD.

\begin{itemize}
  \item On établit une {\bf connexion} avec un serveur distant.

  \item Le dialogue est effectué de maničre transparent 
         grâce ŕ un {\bf driver} qui peut ętre chargé dynamiquement.
  
  \item On effectue des requętes SQL via la {|bf connexion}.

\end{itemize}

Simple et puissant\,!

\end{slide}
